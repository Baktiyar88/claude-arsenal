# Tools

Инструменты, которые устанавливаются отдельно и расширяют возможности Claude. Например: дают ему доступ к базе знаний твоего проекта, к базе данных, или к специфичному API. Большинство работают через **MCP** — это стандарт, по которому Claude общается с внешними программами (как USB для устройств).

## Ресурсы

### [graphify](https://github.com/safishamsi/graphify)

**Что даёт:** Строит "карту связей" твоего проекта (код + документация + картинки + видео) — Claude видит какие файлы связаны между собой и не читает всё подряд, когда ищет ответ
**Когда брать:** Если проект большой или разбираешься в чужом коде и хочешь чтобы Claude сразу понимал "где что лежит"
**Для чего лучше:** универсал (особенно большие/сложные проекты и research по незнакомой кодовой базе)
**Установка:** Python + одна команда установки (см. [инструкцию в репо](https://github.com/safishamsi/graphify))
**Автор:** [@safishamsi](https://github.com/safishamsi)

> **Простыми словами как это работает:** обычный Claude читает твои файлы один за другим, когда ты что-то спрашиваешь. Graphify заранее проходит по всему проекту и строит "индекс" — как оглавление в книге, только для кода: какая функция какую вызывает, какой класс где лежит, какой документ относится к какому коду. Поддерживает **25+ языков программирования** (JavaScript, Python, Go, Rust, Java и другие). После установки Claude через этот индекс мгновенно находит нужные файлы вместо того чтобы перечитывать весь проект.
>
> **Пример пользы:** ты открыл чужой большой проект и спрашиваешь "где обрабатывается авторизация?". Обычный Claude начнёт читать файлы по очереди. С graphify он сразу видит в "карте" все места связанные с авторизацией и показывает их.
